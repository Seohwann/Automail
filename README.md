# Gmail 자동 이메일 발송 웹 UI - 설정 가이드

브라우저에서 Google Sheets의 업체 목록을 불러와 템플릿 치환 + PDF 첨부 + Gmail 라벨링까지 자동화하는 도구입니다.

## 전체 흐름
```
Google Sheets (업체명 + 이메일)
    → Flask 웹 UI (브라우저에서 설정 입력)
    → 템플릿 치환 ({업체명} → 실제 업체명)
    → Gmail API (PDF 첨부 + 발송 + 라벨 자동 적용)
```

---

## Step 1: Python 설치 확인

Python 3.9 이상이 필요합니다. 터미널에서 확인:

```bash
python --version
```

설치되어 있지 않다면 [python.org](https://www.python.org/downloads/) 에서 다운로드.

---

## Step 2: 필요한 라이브러리 설치

터미널에서 아래 명령어 실행:

```bash
pip install flask google-auth google-auth-oauthlib google-api-python-client
```

---

## Step 3: credentials.json 확인

`credentials.json` 파일이 `app.py`와 **같은 폴더**에 있는지 확인하세요.
(이 파일은 압축에 함께 포함되어 있습니다. 별도로 발급받을 필요 없음)

> 본인이 Test users 목록에 등록되어 있어야 로그인할 수 있습니다.
> 등록이 안 되어 있다면 압축을 보낸 사람에게 요청하세요.

---

## Step 4: Google Sheets 준비

스프레드시트를 아래 형식으로 만들기 (**2개 열만 사용**: 업체명, 이메일):

| A (업체명)   | B (이메일)              |
|--------------|-------------------------|
| CJ제일제당   | sponsor@cj.net          |
| 오리온       | promo@orion.co.kr       |
| 빙그레       | event@bing.co.kr        |
| 농심         | collab@nongshim.com     |
| 롯데웰푸드   | partner@lotte.co.kr     |

- 한 칸에 이메일 여러 개도 가능 (쉼표/공백/줄바꿈으로 구분)
- 스프레드시트 URL에서 ID 복사:
  `https://docs.google.com/spreadsheets/d/[이 부분이 Spreadsheet ID]/edit`
- 스프레드시트 공유 설정에서 **로그인할 Google 계정에 뷰어 이상 권한** 부여

---

## Step 5: 실행

`app.py`가 있는 폴더에서 터미널 실행:

```bash
python app.py
```

성공하면 아래와 같이 출력됩니다:
```
==================================================
  Gmail 자동 발송 시스템 웹 UI
  로컬 접속:  http://localhost:5000
  외부 접속:  http://<로컬IP>:5000
==================================================
```

브라우저에서 **http://localhost:5000** 접속.

- 최초 실행 시 브라우저가 열리며 Google 계정 로그인 요청이 뜹니다
- 로그인 후 `token.json`이 자동 생성됨 (다음부터 자동 인증)
- "이 앱은 Google에서 확인하지 않았습니다" 경고가 뜨면 **고급 > 안전하지 않은 페이지로 이동**

---

## Step 6: 웹 UI에서 설정 입력

브라우저에 뜨는 화면에서 아래 항목을 입력:

| 항목 | 설명 | 예시 |
|------|------|------|
| Spreadsheet ID | 4단계에서 복사한 ID | `1AbCdEf...` |
| Sheet Range | 시트 이름 + 범위 | `Sheet1!A2:B100` |
| 제목 템플릿 | `{업체명}` 자동 치환 | `[{업체명}] 제안서 송부의 건` |
| 본문 템플릿 | `{업체명}` 자동 치환 | 자유롭게 작성 |
| PDF 첨부파일 | (선택) 파일 업로드 | `제안서.pdf` |
| Gmail 라벨 | (선택) `/` 로 계층 구분 | `2026 대동제/서환` |

---

## Step 7: 테스트 → 실제 발송

**테스트 모드 (기본 활성화):**
- 스프레드시트의 첫 번째 업체명을 가져와서, 지정한 테스트 이메일로 **1통만** 발송
- 발송 형식이 의도대로 나오는지 본인 메일로 먼저 확인

**실제 발송:**
1. 테스트 결과를 확인한 뒤 **테스트 모드 토글 OFF**
2. **발송 시작** 버튼 클릭
3. 진행 상황이 실시간 표시됨 (성공/실패 카운트)
4. 중간에 멈추고 싶으면 **발송 중단** 버튼 (이미 발송된 메일은 취소 불가)

---

## 주의사항

- **테스트 모드로 본인 이메일에 먼저 보내본 뒤** 실제 발송하세요
- Gmail 일일 발송 한도: 일반 계정 **500통/일** (초과 시 24시간 발송 차단)
- `credentials.json`, `token.json`은 **GitHub 등 공개된 곳에 올리지 마세요**
- `token.json`은 본인 Gmail 인증 정보입니다. **다른 사람에게 공유하지 마세요**
- OAuth 앱이 "테스트" 상태이므로 `token.json`은 **7일 후 만료**됩니다. 만료되면 삭제 후 재실행하면 다시 로그인 창이 뜹니다

## 폴더 구조 (실행 후)

```
app/
├── app.py              # 메인 스크립트
├── credentials.json    # Google OAuth 인증서 (Step 3에서 다운로드)
├── token.json          # 첫 실행 시 자동 생성
├── setup.md            # 본 가이드
└── uploads/            # PDF 업로드 시 자동 생성
```
